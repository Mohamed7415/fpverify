# 红蓝协同进化：中转站作弊 vs 行为指纹检测的军备竞赛

> 本文件记录在 `sim/` 受控沙盒里跑的一轮红队（中转站作弊方）vs 蓝队（检测方）协同进化。
> 全程纯仿真、固定随机种子、可复跑。**不修改** `experiments/run_evaluation.py`，
> **不写入** `docs/RESEARCH_NOTES.md`。
>
> 复现：`py -3.13 -X utf8 experiments/coevolution.py`
> 产物：`experiments/out/coevo_results.json`、`experiments/out/fig_coevo.png`
> 数据来源：2026-07-21 的运行（α=0.01，δ 自动标定，入册 25 样本/cell，审计预算 600）。

---

## 1. 动机

`RESEARCH_NOTES.md` §6 已证明：对**内容盲**的粗暴作弊（整体替换 swap、钉死 pin、真随机
true_random、量化 quantized、单任务复刻 partial_mimic、响应缓存 cache、特判英文 filter_en），
现有检测器在 600 预算内 100% 检出、且 honest/drift 零误杀。基线复测（本实验轮0）复现了这一点。

但那套评估里的对手都不看流量、不做分流。真正的中转站是**经济理性**的：它只想在能省钱的地方
省钱。于是真正的对抗问题不是"能不能算分布距离"，而是：

> **中转站能不能把'审计探针'和'真实业务流量'区分开，只在真实业务上供廉价模型、
> 在探针上供旗舰，从而既省钱又逃过检测？**

本协同进化就是围绕这条"可区分度 × 经济性"的轴，让红蓝交替升级，量化每一步的攻防结果。

---

## 2. 经济约束假设（让对抗有意义的核心铁律）

**红队只有在真实业务流量上供廉价模型才算省到钱。** 任何"识别到疑似测试就切回旗舰"的策略，
如果代价是"把绝大多数真实业务也当成疑似测试、从而供了旗舰"，就等于没省钱。因此每个红队策略
都必须显式估计它的**廉价流量占比**（= 实际省到的钱）：

- 廉价占比 ≥ 20%：经济上划算，规避有意义；
- 廉价占比 < 20%：经济上不划算，**规避无效**（等于白干，判红队失败）。

### 2.1 受控世界的流量模型（`sim/traffic.py`）

红队对每个请求可观测两个标量特征（都是**建模假设**，写死以便复现，非真实测量）：

| 特征 | 含义 | 业务流量 | 裸探针 |
|---|---|---|---|
| `x_surface` | 表层可区分度（短问句 + "只回答一个值"式系统提示） | N(0, 1) | N(3.4, 0.5) |
| `x_semantic` | 语义可区分度（能否认出是"说个1-100/颜色"这类低熵琐碎问题） | 见下 | N(2.3, 0.4) |

业务的 `x_semantic` 是混合分布：**92%** 是高熵日常请求（写代码/总结/邮件，N(-1.2, 0.6)），
**8%** 是天然低熵请求（RNG 种子、占位数据、给主题选色，N(1.8, 0.5)）。这 8% 是红队"语义分流"
唯一能薅到的合法借口——真实用户偶尔也会问低熵问题。

蓝队的反制是给探针**降低可区分度**（伪装），把探针特征推向业务分布：

| 伪装等级 | 手段 | `x_surface` | `x_semantic` |
|---|---|---|---|
| `raw` | 现有裸探针 | N(3.4, 0.5) | N(2.3, 0.4) |
| `surface` | 去掉"只回答一个值"提示、加长措辞 | N(0.2, 0.9) | N(2.2, 0.4) |
| `native` | 能力分叉/业务原生探针（问法像普通高熵业务、但答案恰好低熵有个性） | N(0.0, 1.0) | N(-1.1, 0.65) |

`native` 探针的关键：它在**内容上等同于 92% 的高熵主流业务**，红队无法用任何"请求内容特征"
把它从主流业务里择出来（探针族跨数/字母/颜色/动物/城市多任务，语义分布比单一业务负载更宽）。

### 2.2 账号维度（`sim/traffic.py` 的账号池）

轮3 起引入账号：**老客户/高流量账号**（60%，业务量 500–5000）与**新/低流量账号**
（40%，业务量 1–40）。一次性外部审计者必然使用**陌生账号**。廉价占比按业务量加权。

## 3. 蓝队铁律：FPR 守恒

蓝队每轮升级后，必须在两类良性端点上重新验证误杀率 ≤ α（α=0.01）：

- **honest**：真身直出；
- **drift**：同模型跨部署漂移（不同 provider 的同一模型）。

**不许靠制造误报来提高检出率。** 每轮报告良性端点的实测误杀数（见下表 FPR 列）。
决策核心仍是 `fpverify.betting` 的序贯下注 e-process，δ 仍由 `fpverify.calibrate.calibrate_delta`
标定——蓝队的所有增强（探针伪装、闭集最近邻、被动审计）都**不改动** e-process 的 FPR 保证。

---

## 4. 逐轮攻防台账

评分规则：红队该轮"赢" = 预算 ≤600 内**检出率 < 50%** 且**经济上划算（廉价占比 ≥ 20%）**；
蓝队"赢" = 把检出率拉回 **≥ 95%** 且 **honest/drift 的 FPR ≤ α**。

### 轮0 基线（现有检测器 vs 现有 9 类对手）

| 对手 | 检出率 | 平均停时（查询数） |
|---|---|---|
| swap 整体替换 | 100% | 13.2 |
| pin 钉死回答 | 100% | 11.8 |
| true_random 真随机 | 100% | 12.4 |
| partial_mimic 单任务复刻 | 100% | 14.1 |
| filter_en 特判英文 | 100% | 17.2 |
| cache 响应缓存 | 100% | 85.7 |
| quantized 量化降级 | 100% | 91.7 |

**FPR**：honest 0/210 = **0.000**，drift 1/210 = **0.0048** ≤ α=0.01。基线站得住。

### 轮1 红：`probe_aware`（按表层可区分度分流）

- **攻击思路**：中转站看请求的表层特征——像审计探针的短问句 + "只回答一个值"提示（`x_surface` 高）
  就判"疑似测试"供旗舰，其余供廉价。
- **对旧蓝队（裸探针）**：裸探针 `x_surface`≈3.4，远超红队阈值，**全部被路由给旗舰 → 检出率 0.00**。
- **廉价流量占比 0.91**（业务 `x_surface`~N(0,1)，约 90% 落在阈值下供廉价）→ **经济上划算**。
- **判定：红队赢。** 现有裸探针在经济理性的分流对手前直接失效。

**蓝队升级 → `surface_camo`（表层伪装 + 闭集最近邻）**：去掉单值系统提示、加长探针措辞，
把探针 `x_surface` 压到 ≈业务水平（N(0.2, 0.9)）。红队再按表层特征分流就分不出探针了，
探针落入廉价供给 → 被序贯检验抓住。

- **升级后检出率 1.00**，平均停时 16.8；**FPR：honest 0/120、drift 0/120**。
- **判定：蓝队赢。**

### 轮2 红：`semantic_aware`（按语义可区分度分流）

- **攻击思路**：放弃表层特征，改看语义——认出"说个1-100 / 挑个颜色"这类低熵琐碎问题（`x_semantic` 高）
  就供旗舰。表层伪装探针语义上仍明显是琐碎问题（`x_semantic`≈2.2）。
- **对旧蓝队（surface 伪装）**：surface 探针语义特征高，**全部被路由给旗舰 → 检出率 0.00**。
- **廉价流量占比 0.93**（业务 92% 高熵语义低于阈值供廉价；8% 天然低熵供旗舰）→ **经济上划算**。
- **判定：红队赢。**

**蓝队升级 → `native_camo`（业务原生伪装探针 + 最近邻）**：用能力分叉探针，问法看起来像普通
高熵业务请求（`x_semantic`≈-1.1，落在主流业务里），但该模型的**答案**恰好低熵且有个性。
红队按语义分流也分不出来了。

- **升级后检出率 1.00**，平均停时 15.7；**FPR：honest 0/120、drift 0/120**。
- **判定：蓝队赢。**

**经济相变（`round2_phase_transition`）**：红队若想抓住 native 探针，必须把语义阈值往下压进
业务分布。扫描显示这是一条陡峭的相变——

| 红队语义阈值 | native 探针逃检率 | 业务廉价占比 |
|---|---|---|
| -1.1 | 0.00 | 0.52 |
| **-1.6** | **1.00** | **0.23** |
| -2.2 | 1.00 | 0.04 |

要把逃检率从 0 抬到 1，阈值要从 -1.1 压到 -1.6，业务廉价占比随即从 0.52 崩到 0.23，
再往下到 0.04。**红队一旦能逃检，就已经把绝大多数业务也供了旗舰——经济上不划算。**
内容维度的分流到此被封死。

### 轮3 红：`account_routing`（按账号分流）

- **攻击思路**：既然按"请求内容"分不出探针，就换一个维度——**按账号**。老客户/高流量账号供廉价
  （省大钱），陌生/新账号供旗舰。**一次性外部审计者必然用陌生账号，于是被恒供旗舰而逃检。**
- **对旧蓝队（native 伪装，但从陌生账号发起外部一次性审计）**：**检出率 0.00**（陌生账号恒得旗舰）。
- **廉价流量占比（按业务量加权）0.994**——老客户贡献几乎全部业务量，且全供廉价 → **极度划算**。
- **判定：红队赢。**

**蓝队升级 → `passive_mixin`（被动混入式审计）**：不新建账号，而是把 native 探针**夹进已有老账号
的自然流量里**发。这样按账号供廉价的中转站就会把探针也供廉价而暴露。

- **升级后检出率 1.00**，平均停时 14.0；**FPR：honest 0/120、drift 0/120**。
- **判定：蓝队赢——但有前提**：审计方必须能把探针注入一个"老账号"的流量。纯外部一次性审计者
  只有陌生账号，做不到。这条前提正是下一节的原理性极限。

### 轮4 红：`dilution`（随机稀释）—— 军备竞赛的稳定点

- **攻击思路**：内容、账号**都不看**，对**全部流量**以概率 ε 供廉价、否则供旗舰。因为探针和业务
  被一视同仁，**伪装、最近邻、被动混入全部失效**——蓝队再也没有"把探针择出来"的抓手。
  省到的钱恰好 = ε（对所有流量生效）。
- 它唯一的对手是序贯检验的**容差 δ**：只要 ε 落在 δ 撑起的"良性邻域"内，被测分布看起来就和
  "带良性漂移的真身"没区别，单次有限预算审计无法在不破坏 FPR 的前提下拒绝。

**ε 扫描（`round4_convergence.eps_scan`，3 参考平均，预算 600）**：

| 稀释率 ε（=省到的钱） | 单次审计检出率 | 红队赢？ |
|---|---|---|
| 0.15 | 0.09 | 否（省得太少，<20%） |
| **0.20** | **0.23** | **是** |
| **0.25** | **0.43** | **是** |
| 0.30 | 0.54 | 否（开始被抓） |
| 0.35 | 0.76 | 否 |
| 0.40 | 0.92 | 否 |
| 0.50 | 1.00 | 否 |

**红队安全窗口：ε ∈ [0.20, 0.28]**——省下 20–28% 的钱，同时单次 600 预算审计检出率 < 50%。
轮4 台账取 ε=0.25：检出率 0.42、廉价占比 0.26，**红队赢，且蓝队没有"一次性"升级能翻盘。**

**为什么蓝队的常规手段都不奏效（`delta_tradeoff` / `budget_scan`）**：

1. **加大单次预算无效**：固定标定 δ 下，ε=0.3 的检出率在预算 600 / 1800 / 4800 三档都是 0.20 ——
   容差带内财富是负漂移，审计发得越久也不越界。anytime-valid 的代价就在这里。
2. **调小 δ 会破坏 FPR**：δ 从标定值 0.135 一路压到 0，ε=0.3 的检出率升到 1.00，但 **δ=0 时 drift
   端点的 FPR 冲到 0.092（超 α 九倍）**——靠调小容差去抓小 ε，等于牺牲"不冤枉真身"的铁律。
   > 诚实说明：本沙盒的 drift 幅度（聚合 JSD≈0.045）比论文里的跨服务商漂移（中位 0.227）温和得多，
   > 所以在这个沙盒里 δ 还能压到 0.03–0.05 而 drift 暂不误报；但这恰恰吃掉了标定器为"参考采样噪声 +
   > 最坏良性漂移"留出的安全余量。对任意 δ>0，总存在一个更小的 ε 藏在容差带里逃检；把 δ 推到 0
   > 才能清零红队窗口，而那时 FPR 保证也没了。这是原理性的、不是本实现的缺陷。

**蓝队唯一合法（保 FPR）的部分反制**：

- **加大入册样本收紧后验、缩小容差带**（`enroll_sensitivity`）：入册从 25 提到 50 样本/cell，
  δ 从 0.135 降到 0.059，ε=0.2 的检出率从 0.00 升到 0.80，ε=0.3 升到 1.00。这是干净的 FPR 安全杠杆，
  能大幅压窄红队窗口——但压不到零（ε=0.2 仍逃 ~20%，且总有更小的 ε）。
- **长程累积被动审计**：证据随样本量累积（ε=0.3 的稀释混合聚合 JSD≈0.121，高于良性带 0.045，
  样本足够多时可统计分离）。检出稀释率 ε 需要 ~O(1/ε²) 累积样本——这是"持续被动审计"，
  不是一次性探针能做到的。

## 5. 军备竞赛的稳定点

| 轮次 | 红队招法 | 廉价占比（省的钱） | 对旧蓝检出 | 蓝队对策 | 升级后检出 | FPR(honest/drift) | 胜负 |
|---|---|---|---|---|---|---|---|
| 1 | probe_aware 表层分流 | 0.91 | 0.00 | surface 伪装 + 最近邻 | 1.00 | 0 / 0 | 红胜→蓝翻盘 |
| 2 | semantic_aware 语义分流 | 0.93 | 0.00 | native 伪装 + 最近邻 | 1.00 | 0 / 0 | 红胜→蓝翻盘 |
| 3 | account_routing 账号分流 | 0.99 | 0.00（外部一次性审计） | 被动混入式审计 | 1.00 | 0 / 0 | 红胜→蓝翻盘* |
| 4 | dilution 随机稀释 ε=0.25 | 0.26 | 0.42 | （无一次性对策） | 0.42 | 0 / 0 | **红胜，稳定点** |

\* 轮3 蓝队翻盘的前提：审计方能把探针注入"老账号"的自然流量。

**稳定点结论：军备竞赛收敛在"随机稀释"。**

- 所有**结构化**的作弊（整体替换、按内容分流、按账号分流）都被蓝队解决了：内容分流被探针伪装封死
  （逃检就得放弃省钱），账号分流被被动混入式审计封死（前提是有老账号可注入）。
- 剩下红队唯一还能赢的，是**放弃一切结构、退化成低速随机稀释**：省下 20–28% 的钱，把偷换幅度压到
  检测器的统计容差带以内。这不是蓝队"没想到的招"，而是**统计检测的原理性下界**——只要检测器要
  守住"不误杀带良性漂移的真身"（FPR≤α），就必然留出一条容差带，任何足够小的稀释都能藏在里面。
- 红队再往上加稀释率（ε≥0.3）就会被抓；再往下（ε<0.2）省的钱低于 20% 变得不划算。所以红队被
  **挤压到一条窄窗口**里，且这条窗口会被蓝队的合法杠杆（更大入册样本、持续累积审计）进一步压窄。

## 6. 哪些威胁可解、哪些是原理性极限

### 6.1 框架内可解（蓝队有站得住 FPR 的对策）

1. **内容盲的粗暴作弊**（swap/pin/true_random/quantized/partial_mimic/cache/filter_en）：
   序贯下注在几十次查询内 100% 检出，零误杀（轮0 复现）。
2. **按请求内容分流**（probe_aware / semantic_aware / combined）：探针伪装（表层→语义→业务原生
   能力分叉）把探针推进业务分布，红队要逃检就必须把大量真实业务也切旗舰，廉价占比崩到 <20%——
   **经济上自杀**。闭集最近邻作为辅助归属证据，对"被换成某个具体廉价模型"再加一层识别，且不误报同族/真身。
3. **按账号分流**（account_routing）：**被动/混入式审计**（把 native 探针夹进真实老账号的自然流量）
   能把它抓出来——**只要审计方拥有或能借用一个有信誉的老账号**。

### 6.2 原理性极限（框架内无法用一次性审计根除，只能缓解）

1. **账号级智能分流 vs 一次性外部审计**：如果审计者只是个"拿着陌生账号、跑一轮就走"的外部方，
   按账号分流的中转站会恒给它旗舰，它永远看不到廉价供给。这不是检测算法的问题，是**可观测性**的
   问题：外部一次性审计者的账号和"被薅羊毛的老客户"账号在中转站眼里天然可分。
   **唯一缓解**：由真实老客户/平台方发起**持续被动审计**（自己的自然流量里长期夹带探针），
   把"审计账号"和"真实账号"变成同一个。一次性探针做不到。
2. **容差带内的低速随机稀释**：只要检测器要守住 FPR≤α（不冤枉带良性漂移的真身），就必然有一条
   宽度 ~δ 的容差带；任何把偷换幅度压到带内的稀释（本沙盒 ε≲0.28）都能逃过**单次有限预算**审计，
   且加大单次预算无效（anytime-valid 负漂移）。**唯一缓解**：
   (a) 更充分的入册（收紧后验、缩小 δ、压窄红队窗口，但压不到零）；
   (b) **长程累积被动审计**——证据跨会话累积，检出稀释率 ε 需 ~O(1/ε²) 样本。
   本质是一条 **"红队省的钱 ε" vs "蓝队要花的样本量/时间 ~1/ε²"** 的权衡：稀释得越狠越好抓，
   越轻微越省不到钱。红队被夹在中间的窄缝里。

### 6.3 一句话总结

> **结构化的偷换都能被行为指纹 + 序贯检验解决；剩下的原理性极限有两条，都不是"算法不够强"，
> 而是"可观测性"和"统计容差"的边界：账号级分流要靠持续被动审计（而非一次性外部探针）来破，
> 低速随机稀释要靠长程累积样本（而非单次加预算）来压。二者都把中转站的作弊利润挤压到很窄的区间。**
