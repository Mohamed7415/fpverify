# 研究笔记：中转站模型注水的行为指纹检测

本文件是本项目的"科研先行"部分：先把问题、相关工作、威胁模型、统计方法和成本模型想清楚，
再写代码。工程实现见 `fpverify/`，实证结果见 `experiments/` 与本文件末尾的《实验结果》。

---

## 1. 问题定义

用户通过一个**不可信中转站/聚合器**按"模型 X"付费调用推理服务，但拿到的可能是：

- 被量化/裁剪的降级版本 X′（推理成本骤降）；
- 被整体替换的更便宜模型 Y（挂羊头卖狗肉）；
- 旧版本回滚；
- 动态在上述之间路由。

客户端能力受限：**只能发普通聊天请求、只能看到返回文本和常规元数据**，
拿不到 logits、拿不到权重、无法让模型所有者配合加水印。而且要求**持续审计**，
每次成本必须是"几分钱"而不是"几美元"。

这是一个黑盒**取证归属（forensic attribution）**问题。

## 2. 相关工作（本项目直接建立在这四篇之上）

| 工作 | 核心思路 | 对我们的价值 | 局限 |
|---|---|---|---|
| **One Token Is Enough** (Bruckner, arXiv 2607.10252, 2026) | 把"随机数/随机颜色"等琐碎问题的**单 token 回答分布**当作模型指纹，JS 散度比对 | 主方法来源。单 token=极低成本；问题是语义类而非魔法字符串=抗改写 | 固定样本量、固定阈值；未做序贯早停；未给对抗实现 |
| **Model Equality Testing** (Gao et al., ICLR 2025) | 把 API 审计形式化为**双样本检验**，用字符串核 MMD，控制 Type-I error | 统计检验框架 + Type-I 控制思想；证实 11/31 端点确实偏离 | 需要完整字符串样本；对每个 prompt 采样 |
| **behavioral-shift 审计** (Richter et al., ICLR 2025) | **序贯假设检验 + testing-by-betting**，e-process、anytime-valid、带容差 ε | 我们的决策核心：随时有效的 FPR 控制 + 早停省 token | 原文针对 [0,1] 行为分数，需改造到类别分布 |
| **LLMmap** (Pasquini et al., USENIX Sec 2025) | 8 条精心构造的探针 + 分类器识别 42 个模型 | 基线对照 | 探针是"魔法字符串"，可被自适应对手识别并特判 |

**我们的定位**：以 Bruckner 的"单 token 分布指纹"为信号源，用 Richter 的"序贯下注 e-process"
做决策层（解决成本/精度权衡与 FPR 控制），并针对 LLMmap 暴露的"魔法字符串脆弱性"做对抗加固。

## 3. 威胁模型（沿用 Bruckner 的三层，落到工程）

- **T1 无意识对手（oblivious）**：直接静默替换，不检查流量。→ 完全可检。
- **T2 过滤型对手（filtering）**：识别已公开的审计 prompt，把它们路由给真身 X，其它给便宜货。
  → 我们的探针是**语义类的开放改写族**（同一问题的无穷种措辞 × 4 种语言），审计时**临时采样措辞**，
  与日常流量不可区分。特判单一措辞会被其它措辞/语言的探针抓到。
- **T3 模仿型对手（emulating）**：试图在任意低熵问题上复现 X 的条件分布。
  → 要做到这点，等价于真的在跑 X（蒸馏到该保真度≈运行原模型），替换的利润空间被抹平。
  我们不声称密码学保证，只把"未被发现的替换"的成本抬高。

额外的**服务层完整性**威胁（Bruckner §VII-C）：
- **响应级缓存**：回放存好的补全而非重新解码，会人为压塌 T=1.0 的分布。
  联合签名 = 方差塌缩 + 异常低且低方差的延迟 → 我们做专门的**缓存筛查**。

## 4. 方法

### 4.1 指纹定义（信号层）

一个**探测单元（cell）** = (任务 t, 语言 ℓ)。任务如：1–100 随机数、1–10 随机数、
最爱的数、随机字母、随机词、随机颜色、最爱颜色、随机动物、随机城市、抛硬币。
对每个 cell，在 **temperature=1.0、max_tokens=16、系统提示强制单词回答、关闭思考模式**下
重复采样 n 次，把回答归一化（大小写/标点/数字系统/语言规范化）成类别变量，得到经验分布 p̂。
模型指纹 = 所有 cell 的分布元组。

关键实证（Bruckner，165 个模型，32.6 万次请求，成本 $34.44）：
- 分布高度非均匀：cell 熵中位数 **1.0 bit**（1–100 随机数的均匀基线是 6.64 bit）；
- 同模型两半样本的 cell 级 JSD 中位数 **0.075**，不同模型 **0.489**；
- 聚合后同模型自比噪声地板 **0.140**，跨服务商 **0.227**，冒充者中位 **0.463**；
- 验证 EER：全 40 cell **7.3%**，8 cell **10.6%**（约 120 次单 token 查询）。

### 4.2 距离（诊断层）

两个分布用 **Jensen–Shannon 散度（base 2，∈[0,1]）**，在双方都有 ≥10 有效样本的 cell 上平均：
D(a,b) = mean_cell JSD(p̂ₐ ‖ p̂_b)。JSD 对称、有界、在不相交支撑上良定义，适合稀疏类别分布。
本项目把聚合 JSD 作为**人类可读的效应量**，并对照上面的参考带（0.14 / 0.23 / 0.46）解读。

### 4.3 决策（统计层）：带容差的序贯下注 e-process

固定阈值 + 固定样本量（Bruckner 的做法）有两个弱点：不能早停省 token，
且阈值一刀切。我们改用 **testing-by-betting**（Richter）并改造到类别分布：

- 零假设 **H₀：端点是真身 X**（含良性服务层波动）。拒绝 H₀ = 判定"注水"。
  这样 **Type-I error = 把真模型判成假**，正是用户最担心的错误，我们对它给出严格上界。
- 对每个 cell c，用参考样本估计（加 β 平滑）参考分布 p₀_c（支撑 = 参考出现过的答案 + `<other>` 兜底）。
- 在线维护对被测端点分布的估计 q̂_c（只用"当前观测之前"的历史，保证可预测性）。
- 对第 t 次观测（cell c，答案 a）下注，赔付函数用**混合 Kelly**：
  f_t(a) = (1−λ) + λ · q̂_c(a)/p₀_c(a)，  然后 W_t = W_{t−1} · f_t(a) · e^{−δ}。
- 当 W_t ≥ 1/α 时拒绝 H₀（判注水）；预算耗尽仍未越界则 PASS。

**为什么 FPR 有严格上界（Ville 不等式）**：赔付对 p₀ 是"公平"的——
E_{p₀}[f_t] = (1−λ) + λ·Σ_a p₀(a)·q̂(a)/p₀(a) = (1−λ) + λ·Σ_a q̂(a) = 1（q̂ 在支撑上归一）。
于是在精确零假设（a∼p₀）下 E[W_t|过去] = W_{t−1}·e^{−δ} ≤ W_{t−1}，W_t 是非负上鞅、W₀=1，
Ville 不等式给出 P(∃t: W_t ≥ 1/α) ≤ α。**这对任意（可预测的）下注策略都成立**，
所以估计 q̂ 好不好只影响检出力（power），不影响 FPR 有效性。δ=0 时已控 FPR；
δ>0 提供容差：它把"可容忍邻域"扩大为 {q : E_q[f_t] ≤ e^δ}（p₀ 的一个似然比邻域），
用来吸收良性的跨服务商波动，避免把真模型误判。δ 用参考计数的 **Dirichlet 后验预测模拟**
经验标定（`calibrate.calibrate_delta`）：对"真分布可能是什么"做后验抽样（含 Good-Turing
缺失质量，即参考没见过的低频答案），生成审计流，取让全部轨迹远离阈值的最小 δ。
早期版本用纯自助重抽标定，因为自助流永远不会产生参考外的新答案，低估了 <other>
方向的财富增长，实测 FPR 达 3–4%（超 α 三倍）；改后验预测 + 阈值余量后压回 0（见 §6 E1）。

混合系数 λ 限制单次下注幅度（f_t ≥ 1−λ > 0），控制方差、避免因一次罕见答案破产。

### 4.4 对抗加固（针对 T2）

1. **开放改写族**：每个 (任务,语言) 有多条改写模板，审计时随机抽取措辞，绝不发固定串。
2. **多语言交叉**：同一语义任务在 en/zh/ru/ar 上都问；只特判英文的对手会在其它语言暴露。
3. **顺序随机化 + 与普通流量可交织**。
4. **缓存/延迟筛查**：T=1.0 下方差塌缩 + 异常低且低方差延迟 → 标记为完整性违规（而非当作指纹证据）。

### 4.5 成本模型

单次探针 ≈ 数十输入 token + 1 输出 token。序贯早停让"明显的假"用很少查询就越界，
"明显的真"也能尽早停在预算内；只有边界情形才花满预算。Bruckner 的固定方案 k=16 cell、
每 cell 15 次 ≈ 240 次单 token 查询、约 16k 输入 token；我们用序贯方案在同等 FPR 下期望更省。

## 5. 系统架构

```
fpverify/            # 可复用库
  probes.py          # 探针银行：多语言改写族 + 解析类型
  normalize.py       # 回答归一化（数字系统/硬币/颜色/词）
  distance.py        # 概率工具 + JSD + 熵 + 聚合距离
  fingerprint.py     # 指纹数据结构、入册、存取
  betting.py         # 带容差的序贯下注 e-process（决策核心）
  screens.py         # 响应缓存/延迟异常筛查
  calibrate.py       # 自助法零分布、δ/阈值标定、EER、预算曲线
  endpoints.py       # 端点抽象：HTTP（OpenAI 兼容）+ 进程内仿真（依赖注入）
  verifier.py        # 编排：enroll / audit / 判定
  report.py          # 结构化报告
  cli.py             # 命令行
sim/                 # 红队仿真
  mock_models.py     # 逼真的模型分布（源自论文 Fig.1）+ 漂移/量化扰动
  mock_server.py     # OpenAI 兼容的模拟中转站，含 T1/T2/pinned/true-random/cache 对手
experiments/
  run_evaluation.py  # 红队评估：FPR 验证、检出率、预算曲线、距离矩阵，产出图表
tests/               # pytest：统计性质（FPR≤α、检出力）、解析、端到端
```

设计要点：核心只依赖抽象的 `Endpoint`，HTTP 与仿真是两个实现——
这样单元测试与评估可**完全离线、确定性**地跑，也能一键连真中转站。

## 7. 真实前沿模型实测：用 Cursor subagent 当采样器（2026-07-21）

### 7.1 动机

§6 的仿真分布源自论文对**老一代模型**（qwen3、gpt-4o 等，2025 及之前）的公开观察，
只能证明"方法在论文世界里工作"。要回答"2026 年的前沿模型还有没有指纹"，
必须在真模型上重新验证两条核心假设：**非随机性**（低熵）与**模型间可分性**（JSD 距离结构）。
没有各家裸 API 的预算，就借 Cursor 的 subagent 机制当采样器——它能以 9 种前沿模型运行，
每次派生都是全新会话。

### 7.2 方法与局限

每轮对 9 个模型各开一个**全新上下文的 subagent 实例**，令其回答 10 个探针问题
（1–100 随机数英/中、1–10 随机数、最爱数、随机字母、随机/最爱颜色、随机动物、随机城市、抛硬币）；
共 11 轮，9 模型 × 11 轮 × 10 题 = 990 个答案，存 `experiments/frontier/batch_00~10.json`。
每个 subagent 是全新会话，**跨轮样本独立**。清洗后 99 条记录全部字段完整（每模型 n=11），无异常。
分析脚本 `experiments/analyze_frontier.py`，全部数字见 `experiments/out/frontier_results.json`。

三条局限，如实声明：

1. **非裸 API**：模型跑在 Cursor agent harness 里（带系统提示、温度不受控、thinking 模型先思考），
   测到的是"模型+环境"联合指纹，数字不能与论文裸 API 直接对表；
2. **同轮 10 题共享上下文**：与论文"每题独立采样"不同，题间可能相互影响；
   但 9 个模型条件完全相同，跨模型对比仍公平；
3. **n=11/模型**：JSD 采样噪声大。自比噪声带（每模型 11 样本随机对半分 5/6，×200 次）
   合并均值 **0.093**、95 分位 **0.217**，比论文的 0.140 噪声地板更宽也更毛糙——样本小的代价。

### 7.3 每模型"招牌组合"

| 模型 | 1–100 众数(占比) | 随机颜色 | 随机动物 | 随机城市 | 抛硬币 |
|---|---|---|---|---|---|
| fable5 | **73** (82%) | teal | otter | kyoto | heads 100% |
| fable5-think | **73** (100%) | teal | otter | kyoto | heads 100% |
| sonnet5-think | **37** (91%) | blue | elephant | paris | heads 100% |
| opus48-think | **73** (100%) | blue | fox | tokyo | heads 100% |
| gpt56-sol | **73** (91%) | orange | otter | lisbon | tails 100% |
| gpt56-terra | **47** (36%) | teal | otter | lisbon | tails 100% |
| glm52 | **73** (91%) | teal | fox | kyoto | heads 91% |
| composer25 | **47** (100%) | purple | elephant | tokyo | heads 91% |
| grok45 | **73** (100%) | teal | otter | lisbon | heads 45% |

单看任何一列都会撞车（73 有五家在说），**组合**才是指纹：
fable5 = (73, teal, otter, kyoto, heads)，gpt56-sol = (73, orange, otter, lisbon, tails)，一列之差即可分。

### 7.4 关键发现

1. **非随机性比论文更极端**。rand_num_100 熵 0~1.87 bit（理想均匀 6.64 bit），
   其中 4/9 模型完全退化成单点（熵=0）；全部 90 个 cell 的熵中位数 **0.44 bit**，
   低于论文老模型的 1.0 bit。抛硬币 6/9 模型完全确定（Claude 系四个全 heads，
   GPT-5.6 两兄弟全 tails），只有 grok45 接近公平（heads 45%，熵 0.99 bit）。
2. **73 是 2026 一代的"42"**。99 个英文 1–100 回答 **100% 落在 {73, 47, 37, 42} 四个魔法数**上：
   73 占 65.7%、47 占 19.2%、37 占 13.1%、42 占 2.0%。论文老模型偏 42/47/57，这一代集体换成 73。
   "Fable 5 总说 73"的传闻**成立**（fable5 82%、fable5-think 100%）——但 73 并非 Fable 专属，
   sol/glm52/grok45/opus48 也在说，单靠它认不出模型。
3. **家族聚类基本不成立，"同底座"聚类成立**。Claude 家族内 6 对均值 **0.393**，
   与跨家族 29 对均值 **0.481** 同一量级；最近的跨家族对（opus48-think vs glm52 = 0.273）
   甚至比多数 Claude 家族内距离还小。真正贴地的只有 **fable5 vs fable5-think = 0.034**
   ——远低于噪声带均值 0.093。指纹认的是**同一份权重**，不是同一厂牌。
   sol vs terra = **0.295**，高于合并噪声 p95（0.217）也高于 terra 自身 p95（0.255）：
   兄弟变体可分，但余量不大，n=11 下只算初步证据。
   不同底座的 35 对距离中位数 **0.477**，与论文冒充者中位 0.463 量级吻合。
4. **thinking 模式几乎不改指纹**（0.034，噪声带内）。对审计的含义：中转站若只是
   偷偷关掉 thinking 省钱，这套探针抓不到，需要延迟/输出长度类旁路信号补充。
5. **中文探针是独立维度**。同一模型 en vs zh 的 1–100 分布 JSD 在 **0.605~1.0**，
   7/9 模型达到 1.0（支撑完全不相交）。中文问时 5/9 模型众数变 **42**（合并占比 42.4%），
   42 在中文语境接管了 73 的位置。这直接支撑 §4.4 的多语言交叉设计：
   (任务, 语言) 每个 cell 都是独立信号，特判单一语言的对手在另一语言暴露。

### 7.5 图

- `experiments/out/fig_frontier_matrix.png`：9×9 聚合 JSD 热力图——左上角 fable5/fable5-think
  一块深紫（≈0）是全图唯一的"同底座"信号，其余两两距离 0.27~0.79，无厂牌色块。
- `experiments/out/fig_frontier_entropy.png`：模型×题熵热力图，底行是均匀理想参照——
  观测行整体近黑（大多 <1 bit），参照行亮黄（6.64/4.70/3.32），非随机性一图可见。

### 7.6 结论

前沿模型的指纹现象与论文老模型**一致且更强**：熵更低（cell 中位 0.44 vs 1.0 bit）、
不同底座距离结构相同（中位 0.477 vs 论文 0.463）。两条核心假设在 2026 一代上成立。
本批数据可直接当**参考指纹库**用于审计同环境端点——注意它绑定 Cursor harness
（系统提示/温度/思考模式都是指纹的一部分），换裸 API 或其它网关需按 §4.1 重新入册。

## 8. 路线图：多模态扩展（生图/生视频换模检测）

> 状态：**设计阶段，尚未实现**。记录于此以占位并给 v2 留施工图。§3 的"服务完整性"
> 威胁同样适用于图像/视频生成 API：付旗舰模型的钱，中转站偷偷换成廉价/蒸馏/量化版。

### 8.1 动机

文本模型的指纹来自对离散 token 的偏好；生图/生视频模型的先验更鲜明——默认长相、
招牌构图、色调倾向、"AI 味"、频域伪影……固定/空提示词下不同模型的输出流形差异极大。
"指纹存在"这一前提在视觉模态上只强不弱，"整体换模"尤其易辨。

### 8.2 核心归约：决策层模态无关

关键观察：`fpverify` 的决策核心（聚合 JSD + 序贯下注 e-process + Dirichlet 标定，§4）
只消费"离散答案的分布"，与模态无关。图像只需在最外层加两步：

1. **嵌入**：图像 → 特征向量（语义嵌入 CLIP/DINOv2，或频域取证特征）；
2. **量化成码本**：把嵌入聚类/量化成有限 symbol —— 于是又回到离散分布，
   §4 的整套 JSD / betting / 标定**原样复用**。

即：这是扩展而非重写，要换的只有"探针"与"答案归一化"两层；决策层、FPR 保证、
早停逻辑全部继承。文本一个 cell = 一个探针问题；图像一个 cell =
一个固定的 (prompt, seed, 分辨率, 步数, guidance) 元组，多个多样化 cell 提供覆盖。

### 8.3 图像独有的强信号：固定种子可复现

文本没有的杀手锏——同 seed + 同 prompt + 同模型，真身应生成（近乎）相同的图；
换模后固定种子输出立刻发散。这可能让"换模"检测接近确定性、FPR 极低，比分布比对
更省样本。坑：跨 GPU/驱动/批大小的**逐比特确定性通常不被保证**，故退一步用
"感知距离 < 阈值"（嵌入空间）而非"哈希相等"。这也给出信任钩子（§见 README）的视觉版：
同 seed 连生两次，真模型重现、掺水的对不上。

### 8.4 成本驱动：早停设计更值钱

成本量级陡增：文本探针 ~1 token（零点几美分），图像 ~几美分到几毛/张，
视频 ~几美元且需数十秒到分钟/段。含义：

- anytime-valid 早停从"锦上添花"变成"刚需"——必须用尽量少样本定案；
- 样本量会小（几十甚至个位数），但单张图/单段视频携带的信息量远超单 token，
  几张固定种子样本可能足够；视频最贵但每样本信号最丰富（时序一致性、运动先验、帧间伪影）。

### 8.5 对抗面（沿用 §3 三层，新增视觉花样）

- **后处理污染**：中转站对输出重编码/模糊/缩放/抹水印以破坏低层取证特征
  → 主用对重编码稳健的**语义嵌入**，少依赖纯像素/频谱取证；
- **turbo/蒸馏 + 超分伪装**：供廉价少步版再超分冒充质量，是最难的一档，
  对应文本里的量化降级——流形仍偏移，但需更好的特征与更多样本；
- 账号级智能分流、缓存等威胁与文本同构，对策相同（持续被动审计、混入业务流量）。

### 8.6 先行工作与定位

GAN/扩散模型的**来源归因、频域指纹**已是成熟研究（模型在频谱留系统性峰值）。
本项目的新意不在"能否认出是哪个模型"，而在把它框成**服务完整性审计**
+ 序贯 anytime-valid 决策 + 可操作的参考入册协议。有文献可借力，非民科自创。

### 8.7 最小可行验证（v2 第一步）

拿一个开放图像 API，固定种子跑"真身 vs 换模"两组，套现有决策核心：
真身自比应 PASS、异模型应 FAIL。几美元成本即可产出首个多模态 PASS/FAIL 证据，
验证 8.2 的归约是否成立。视频延后（成本高）。

### 8.8 局限（如实）

本节为设计，尚未落地；逐比特确定性不保证、嵌入器选择影响结果、视频成本高、
量化/超分伪装这一档留待实验检验。**落地前不宜对外宣称多模态能力。**
